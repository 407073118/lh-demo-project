# LH Quant 架构说明

## 目标

构建一个本地优先的量化研究入门项目，让人能完整跑通：数据契约、信号生成、回测执行、指标展示和
命令行使用。

## 核心边界

- `lh_quant.data` 负责 OHLCV 数据校验和真实数据下载；开发演示数据也必须先入库再使用。
- `lh_quant.storage` 负责数据库表结构、行情覆盖区间、行情缓存和回测运行记录。
- `lh_quant.strategies` 只负责生成信号，不依赖券商或回测框架。
- `lh_quant.backtest` 把 K线和信号转换成成交、权益曲线和汇总指标。
- `lh_quant.cli` 暴露可以直接运行的命令行流程，默认下载入库、从库回测。

## 设计选择

v0.1 使用一个小型内部回测引擎。这样行为容易审计，也不会让第一课被大型框架 API 淹没。外部框架
后续应该作为适配层加入，而不是直接成为核心领域模型。

当前引擎故意保持简单：单标的、只做多、按收盘价成交、固定手续费率、暂不考虑滑点、做空和杠杆。
这些限制是明确写出来的，后续可以逐步变成新的测试和功能。

## Workbench 结果和配置模型

Web 工作台已经把回测配置拆成 Universe/Data、Strategy、Risk/Execution 三组。策略注册表会向 API 暴露参数定义和跨字段约束，前端表单与后端参数归一化共用同一套约束语义，避免策略参数校验散落在多个硬编码分支里。

回测运行现在持久化更完整的指标，包括年化收益、年化波动、Sharpe、Sortino、Calmar、胜率、盈亏比、期望、佣金、仓位暴露和换手率。`/api/backtests/{runId}` 可以按运行编号读取已落库的摘要、权益点、交易记录和信号点；前端最近回测列表也可以打开这些历史运行摘要。
